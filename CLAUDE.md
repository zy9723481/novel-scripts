# novel-scripts 项目配置

## 写小说规则

### 文章质量标准
- 段落使用全角空格「　　」首行缩进，段落之间不留空行
- 标点符号必须多样化：句号、逗号、问号、感叹号、省略号、冒号合理搭配
- 严格控制破折号「——」使用频率，优先用冒号、逗号替代
- 中文网文口语化叙事，短句为主，避免长难句和书面化辞藻

### 伏笔管理
- 每埋一个伏笔/坑，必须同步记录到「伏笔追踪.md」
- 记录内容：伏笔ID、铺设章节、内容、类型、状态（已埋/已揭示/已回收）
- 后续章节回收伏笔时，必须更新伏笔追踪表中的状态
- 禁止只挖坑不填坑、只埋伏笔不解释伏笔

### 合理性校验（强制性）
- **每写完一章后，必须执行全维度合理性校验**：
  1. 对照「世界观与全本大纲」，检查剧情是否符合设定
  2. 对照前文所有章节，检查是否有逻辑漏洞、剧情矛盾、时间线错乱
  3. 对照人物设定，检查是否有 OOC（角色行为不一致）
  4. 检查新写内容中的伏笔是否已记录到伏笔追踪表
  5. 检查是否有前文伏笔可在本章回收
- **校验不合格的章节必须重写，不得跳过**
- 校验通过后才能开始写下一章

### 体量要求
- 单章中文字数 ≥ 3000 字
- 每 3000 字内至少设置 2-3 个爽点/反转点
- 每章结尾必须留强钩子
